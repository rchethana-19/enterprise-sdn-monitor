# SDN NOC Dashboard (OVS-Vision)

A secure, real-time **Software-Defined Networking (SDN) Network Operations Center (NOC)** platform that simulates an enterprise network using **Mininet** and **Open vSwitch (OVS)**. The project features a custom **Flask** telemetry engine and an interactive **D3.js** dashboard for real-time visualization of network topology, switch performance, packet statistics, latency, and firewall status.

To provide enterprise-grade monitoring capabilities, the platform integrates **Splunk Enterprise** through the **HTTP Event Collector (HEC)**, enabling centralized telemetry collection, historical analysis, and laying the foundation for future security event detection and SOC-style monitoring.

---

# Features

* **Enterprise SDN Topology Simulation:** Simulates a multi-department corporate network (Engineering, HR, Finance, Server Zone, and Firewall) using Mininet and Open vSwitch.
* **Real-Time OVS Telemetry:** Polls Open vSwitch instances using `ovs-ofctl` to collect packet statistics (Rx/Tx), dropped packets, and interface metrics.
* **Host Latency Monitoring:** Continuously measures host-to-host latency across the enterprise network.
* **Firewall Status Monitoring:** Tracks firewall availability and basic enforcement status.
* **Interactive D3.js Dashboard:** Displays a live force-directed network topology with real-time performance metrics.
* **REST API Backend:** Flask-based backend exposes telemetry through JSON endpoints for dynamic dashboard updates.
* **Splunk Enterprise Integration:** Streams telemetry to Splunk Enterprise using the HTTP Event Collector (HEC) for centralized logging and analysis.
* **Historical Network Analytics:** Enables searchable telemetry, historical trend analysis, and future security correlation within Splunk.
* **Enterprise Dark Theme UI:** Responsive interface designed to resemble a professional Network Operations Center (NOC).

# Network Architecture

The infrastructure replicates a secure enterprise subnet where departmental switches connect to a centralized core switch protected by an administrative firewall.

```text
               [ Internet ]
                    |
               [ Firewall ] (fw1)
                    |
             [ Core Switch ] (s1)
          /      /      \      \
        s2      s3       s4     s5 (Server Zone)
       /        |        |       | \
   [Eng]      [HR]   [Finance]  Web  DB
  h1,h2,h3   h4,h5     h6,h7    h8   h9
```

# Splunk SIEM Integration

The project integrates **Splunk Enterprise** using the **HTTP Event Collector (HEC)**.

Telemetry generated by the monitoring engine is transmitted as structured JSON events to Splunk, enabling:

* Centralized telemetry collection
* Historical network analytics
* Searchable packet and latency data
* Long-term performance monitoring
* Future security event detection and alerting

Each telemetry event includes:

* Switch statistics
* Packet counts
* Packet loss
* Host latency
* Firewall status
* Timestamp
