# Splunk Integration

OVS-Vision integrates with Splunk Enterprise using the HTTP Event Collector (HEC) to centralize security telemetry and support SOC-style monitoring.

---

## Overview

Security events generated by the monitoring engine are forwarded to Splunk, where they can be searched, visualized, and analyzed using SPL queries and dashboards.

---

## Forwarded Events

OVS-Vision forwards the following information:

- Firewall status
- Packet loss alerts
- High latency alerts
- Security alerts
- MITRE ATT&CK mappings
- SOAR playbooks

---

## Event Fields

Each event includes:

| Field | Description |
|-------|-------------|
| incident_id | Unique incident identifier |
| severity | Alert severity |
| category | Security category |
| title | Alert title |
| description | Alert description |
| affected_device | Device associated with the alert |
| timestamp | Event timestamp |
| mitre | MITRE ATT&CK mapping |
| playbook | Generated SOAR playbook |

---

## Sample Event

```json
{
    "incident_id": "INC-4A9D1B7C",
    "severity": "HIGH",

    "title": "High Packet Loss Detected",

    "mitre": {

        "technique": "Network Denial of Service"
    },
    "playbook": "Network Investigation"

```

## Splunk Dashboards

The following dashboards were created:

- Total Security Events
- Events by Severity
- MITRE ATT&CK Distribution
- Incident Timeline
- Top Affected Devices

---

## Example SPL Queries

### Total Security Events

```spl
index=main sourcetype="ovsvision:security"
| stats count
```

---

### Events by Severity

```spl
index=main sourcetype="ovsvision:security"
| stats count by severity
```

---

### Top Affected Devices

```spl
index=main sourcetype="ovsvision:security"
| top affected_device
```

---

### Incident Timeline

```spl
index=main sourcetype="ovsvision:security"
| timechart span=1m count
```

---

### MITRE ATT&CK Distribution

```spl
index=main sourcetype="ovsvision:security"
| spath
| stats count by mitre.tactic
```
